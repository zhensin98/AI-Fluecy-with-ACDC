Welcome

Welcome to Programme 1: Discover

Welcome to the AI Fluency Roadmap for Software Engineering. This is Programme 1 of 4 — DISCOVER. You are at the starting point of a progressive journey that takes software engineers from AI-aware to AI-native.

What You'll Learn

By the end of this session, you will be able to:

  • Understand the capability threshold crossed in 2024–2025 and why the previous mental model of software development is now outdated
  • Identify the AI coding tools relevant to your role — Claude Code, Cursor, GitHub Copilot — and how they differ in practice
  • Map the 7 New Habits of AI-Native Software Engineering to your daily workflow and role
  • Spot where AI can remove friction from your current process immediately
  • Commit to one habit you will practise in the first week after the session
Audience

All software engineering roles — developers, QA, DevOps, architects, and tech leads.

Duration

Half-day (4 hours)

Format

Facilitated workshop with live tool demonstrations and a habit-mapping exercise.

Objective
Programme Objective

Most engineers know AI tools exist. Few have fundamentally changed how they work. This session creates that break — from passive awareness to active habits. Participants understand what has changed in software engineering since 2024–2025, which tools apply to their role, and which of the 7 New Habits of AI-Native Software Engineering they will build first.

The core shift: From writing code to directing agents that write code — and knowing exactly how to evaluate what they produce.
Key Topics

The session covers the following topic areas. Each builds directly towards your first Claude Code interaction.

1 The Agentic Engineering Thesis

An introduction to the MISSION+ ACDC framing — the Agent-Centric Development Cycle — and why agentic engineering represents a fundamental shift, not an incremental upgrade.

  • What changed in 2024–2025 and why it matters for software teams
  • The previous mental model vs the AI-native mental model
  • Why the bottleneck has shifted from building to specifying
2 The AI Tools Landscape for Software Development

A practical comparison of the tools available to software engineering teams today.

  • Claude Code (CLI and Web) — agentic coding assistant
  • GitHub Copilot — inline code completion and chat
  • Cursor — AI-native IDE
  • How to choose the right tool for your role and workflow
3 The 7 New Habits of AI-Native Software Engineering

An overview of the 7 New Habits framework — the MISSION+ original IP that underpins every programme in this roadmap.

  • SH1: Spec Author — write intent, not code
  • SH2: Agent Orchestrator — direct agents, verify outcomes
  • SH3: Parallel Builder — run multiple agents concurrently
  • SH4: Shift-Left Tester — QA starts at the specification
  • SH5: Vibe, Then Verify — generate fast, govern carefully
  • SH6: Compounding Engineer — encode team knowledge into CLAUDE.md
  • SH7: Night Shift Commander — define what runs overnight
4 Role-Habit Relevance Mapping

Not every habit is equally important for every role. In this exercise, participants map the 7 New Habits to their own daily workflow and identify which habit they will build first.

  • Role tracks: Software Engineers, QA Engineers, Tech Leads, Product & Planning, DevOps & Platform, Security Engineers
  • Priority mapping exercise — which habit to build first
  • Committing to one habit for the first week
5 First Interactions with Claude Code

Participants make their first live interaction with Claude Code during the session — not as a demo, but as a hands-on first step.

  • Installing and launching Claude Code (CLI)
  • Your first Claude Code command
  • What to expect from an agentic coding assistant
