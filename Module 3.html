Welcome

Welcome to Programme 3: Build

Theory stops here. Two days of building — parallel workspaces, self-healing QA agents, MCP integrations, and the full ACDC Lifecycle from specification to staged release. Every participant leaves with working AI workflows deployed to their real project, not a sandbox.

What You'll Learn

By the end of this programme, you will be able to:

  • Set up and manage parallel Claude Code workspaces using the "10 Claudes" strategy for concurrent feature delivery
  • Build and deploy Agent Skills (SKILL.md) that encode team standards and automate repetitive engineering workflows
  • Implement self-healing QA agents using MCP integrations with test runners, Sentry, and Slack
  • Configure CLAUDE.md as an institutional knowledge and governance layer for the engineering team
  • Execute a "50 PRs a Week" challenge — delivering 3–5 features simultaneously through parallel agentic workflows
  • Walk the complete ACDC Lifecycle: Prototype → Specify → Ticket → Build → Validate → External Scan → Release
Audience

Software Engineers, QA Engineers, DevOps and Platform Engineers, and Tech Leads.

Duration

2 full days (role-specific tracks in afternoon sessions)

Format

Lab-based, 100% hands-on. Participants work on real codebases.

Role Tracks

Afternoon sessions are role-specific. Select your track to see what you will build.

A Track A: Developers

DEVELOPERS

Build at speed. Parallel workspaces and the 10 Claudes strategy for delivering multiple features simultaneously.

What You Will Build

  • Parallel workspaces: Run 10 concurrent Claude Code sessions across independent feature branches, delivering multiple features at the same time without context-switching
  • Generative UI: Use Claude to scaffold UI components, APIs, and data models directly from specifications — skipping manual boilerplate entirely
  • 50 PRs a Week challenge: Deliver 3–5 production-ready features simultaneously through parallel agentic workflows, measured and reviewed each day

B Track B: QA Engineers

QA ENGINEERS

Shift left. Self-healing tests and the Bugbot QA agent for autonomous defect detection and resolution.

What You Will Build

  • Self-healing tests: Write tests that automatically adapt when implementation changes — eliminating the maintenance burden that kills test coverage over time
  • Bugbot QA agent: An autonomous agent that detects, reproduces, and triages defects — generating structured bug reports with reproduction steps and suggested fixes
  • Shift-left automation pipeline: Integrate QA agents into the build pipeline before code review, catching regressions at the earliest possible point

C Track C: DevOps / Platform

DEVOPS / PLATFORM

Connect everything. AIOps, MCP integrations, and predictive infrastructure maintenance.

What You Will Build

  • AIOps: Use Claude agents to monitor, diagnose, and resolve infrastructure incidents automatically — reducing mean time to resolution without human intervention for known failure patterns
  • MCP integrations: Connect Claude Code to Sentry, Slack, and test runners via Model Context Protocol, enabling agents to act on real signals from your production environment
  • Predictive maintenance: Agents that analyse logs and metrics to surface failure risks before they become incidents, and draft remediation runbooks automatically

D Track D: Tech Leads

TECH LEADS

Govern the system. Agent Skills authoring, CLAUDE.md governance architecture, and team memory setup.

What You Will Build

  • Agent Skills authoring: Write SKILL.md files that encode team standards and automate repetitive workflows — skills that every developer on the team can invoke immediately
  • CLAUDE.md governance: Configure Claude Code as an institutional knowledge layer for the entire team — architectural decisions, code standards, domain vocabulary, and deployment constraints all in one file
  • Team memory setup: Define persistent context that every agent session inherits automatically, so new sessions start with full awareness of the team's conventions and constraints

AI Night Shift

The final session of Programme 3 is planning your personal AI Night Shift — the automated overnight operation where agents continue working after you log off.

  • What runs overnight: test suites, security scans, dependency updates, and draft PRs for known backlog items
  • Morning triage: a structured review of overnight agent output — approve, redirect, or reject before the team starts work
  • Your Night Shift plan: each participant leaves with a documented plan for their specific codebase and workflow
