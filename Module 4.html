Agentic Engineering at Team Scale

The shift from individual AI productivity to team-level architecture. At scale, the team becomes the agent — and the engineering leader's job is to design the system, not just use the tools.

  • Compound growth: every Agent Skill and CLAUDE.md addition multiplies the output of the whole team
  • Governance as acceleration: clear guardrails make agents faster, not slower
  • Pilot selection: choose the right team and project to demonstrate measurable ROI in 30 days
Pathfinder Governance Agent

The Pathfinder Governance Agent provides continuous compliance monitoring, gap analysis, and remediation guidance across the engineering organisation.

  • Compliance reporting: automated checks against ACDC standards across all active branches
  • Gap analysis: identifies where teams are deviating from agreed specifications
  • Remediation guidance: generates specific, actionable instructions to bring work back into alignment
AI Night Shift Operations

The AI Night Shift is the automated overnight operation where agents continue working after the engineering team logs off — running tests, scanning for issues, drafting PRs, and preparing the morning triage list.

  • What runs overnight: test suites, security scans, dependency updates, draft PRs for known backlog items
  • Who reviews it: the morning triage owner reviews agent output and approves, redirects, or rejects
  • Triage protocol: a structured 15-minute morning review that converts overnight agent work into actionable tasks
CLAUDE.md as Institutional Memory

CLAUDE.md is not a configuration file — it is the team's institutional memory. Every standard, constraint, preference, and lesson learned that should apply to every future agent session.

  • What belongs in CLAUDE.md: architectural decisions, code style standards, domain vocabulary, deployment constraints
  • Governance architecture: org-level, team-level, and project-level CLAUDE.md hierarchy
  • Compounding value: each addition to CLAUDE.md is a permanent improvement to every future session
30-60-90 Day Roadmap

The workshop output is a completed 30-60-90 day roadmap. Not a slide deck — a working document with named pilots, measurable milestones, and a CCA-F certification pathway.

Days 1–30: Pilot
  • Select pilot team and project
  • Deploy CLAUDE.md and first Agent Skills
  • Complete first ACDC Lifecycle end-to-end
Days 31–60: Expand
  • Roll out to 2 additional teams
  • Deploy Pathfinder Governance Agent
  • Launch AI Night Shift operations
Days 61–90: Compound
  • Measure and report productivity baseline vs. outcome
  • Establish CCA-F certification pathway for team
  • Publish internal case study

Three-Pillar Governance Model

1

Complete Specs

Every task begins with a specification that meets the ACDC standard before any agent is invoked.

2

Organisational Guardrails

CLAUDE.md and Agent Skills encode the rules so agents operate within agreed boundaries automatically.

3

Automated Code Review

The Pathfinder agent reviews agent-generated code for compliance before it reaches human review.
