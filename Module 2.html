Objective
Programme Objective

The most common failure mode in agentic development is a vague specification producing vague code. This workshop builds the discipline of spec-first thinking — turning fuzzy requirements into agent-executable instructions, and applying critical thinking to everything the agent produces. The bottleneck is no longer building. It is deciding what to build, and saying it with precision.

The core insight: Specification quality drives output quality. A vague spec produces vague code. A precise spec produces precise code.
Key Topics

This full-day workshop is structured around five topic areas. Each one builds directly into the next.

The TAEO Framework Applied to Engineering

Think-Ask-Evaluate-Own (TAEO) is the MISSION+ original framework for critical thinking in an AI-native workflow. In this programme, it is applied specifically to engineering decisions and AI interactions.

  • Think: What am I actually trying to build? What are the constraints?
  • Ask: How do I frame this for the agent? What inputs does it need?
  • Evaluate: Is this output correct, complete, and safe to accept?
  • Own: I am accountable for what goes into production — not the agent
Prompt Engineering for Code and Specifications

Writing effective prompts for code generation is a different skill from general prompting. This topic covers the principles that make agent instructions executable.

  • The difference between vague and precise specifications
  • Acceptance criteria as agent instructions
  • Edge cases, data model definitions, and constraint language
  • How prompt quality cascades through the Cascade Workflow
Claude Code Plan Mode Deep Dive

Claude Code Plan Mode (activated with Shift+Tab) generates a structured, step-by-step plan from natural language before writing a single line of code. This topic is a hands-on deep dive.

  • Activating and using Plan Mode (Shift+Tab)
  • Reading and evaluating a Level 1 Plan
  • Iterating on the plan before approving execution
  • How Plan Mode enforces spec-first thinking
The Cascade Workflow

The Cascade Workflow is the MISSION+ model that shows how specification quality at each stage determines output quality at the next. It is the practical illustration of why spec-first thinking matters.

  • How vague intent cascades into vague code
  • How precise specs enable reliable agent execution
  • Introduction to ACDC spec structure
  • Identifying where specification quality is the real bottleneck in your current workflow
Exercises on Real Work Tasks

Participants bring real tasks from their own codebase. The workshop exercises apply TAEO and spec-writing directly to those tasks — not toy examples.
